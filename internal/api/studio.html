🔌 Connect to Centauri

Use a zero-dependency client, plain HTTP, or MCP. Insert = update; reads, time-travel, context, and CDC are one call each.

Python — pip install centauri-sdk · /sdk/python
from centauri import Centauri
db = Centauri(token="…")
db.add("toy:robot", {"price_cents": 500})
db.context("toy:robot")            # the agent bundle
db.run("reprice", item="toy:robot", pct=90)
Go — /sdk/go
c := centauri.New(url, centauri.WithToken("…"))
c.Add("toy:robot", map[string]any{"price_cents": 500})
c.Query("FACTS OF item:* WHERE region='EU'")
c.Run("reprice", map[string]any{"item":"toy:robot","pct":90})
JavaScript / TypeScript — /sdk/js
const db = new Centauri(url, {token:"…"});
await db.add("toy:robot", {price_cents: 500});
await db.query("FACTS OF toy:robot WHY");
HTTP — any language
curl -s URL/v1/query -H 'Authorization: Bearer …' \
     -d '{"q":"FACTS OF toy:robot WHY"}'
MCP — agents (zero code)
{ "mcpServers": { "centauri": { "command":"centauri", "args":["mcp"] } } }
Procedures (CePL) — not Python: a tiny in-DB procedure language, run by name with a step trace
PROCEDURE reprice(item, pct)
  LET cur = FIRST FACTS OF ${item}
  WHEN cur IS MISSING: FAIL 'unknown ${item}'
  LET newp = cur.price_cents * pct / 100
  PUT ${item} SET price_cents=${newp}
  RETURN newp
END
-- then:  RUN reprice WITH item='toy:robot', pct=90

🎲 Demo data — learn Centauri on real-shaped data

Loads a curated dataset across retail, healthcare, education & construction into a dedicated demo database — planted with bi-temporal corrections, causal chains, pending "wedges", cross-source disagreements, an AI enrichment, schemas and aggregates. It never touches your real data.

"Clear" drops the entire demo database file — consistent with Centauri's "nothing is ever erased" (that rule protects facts inside a live log, not a disposable example database).

📎 Vision — let an AI see your files

Upload an image or PDF (e.g. an electrical drawing). Centauri stores the blob locally (content-addressed, not in the log), renders PDF pages, then a vision model you register describes it and embeds it into the vector index — searchable with no external store.

1) Register a local vision model (once)

Defaults target Ollama (ollama pull llava). One click writes model:vision (+ an embedder) as facts.

2) Analyze + index

Then search them: SEARCH 'E-3 200A panel' · FACTS OF asset:* · SIMILAR TO <asset>
